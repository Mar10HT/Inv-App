PATTERNS DOCUMENTED
================================================================================

Recent Patterns (Featured in RECENT-PATTERNS.md):

1. Dialog Result Types
   Pattern: Type-safe dialog outputs via input/output decorators
   Files: loan-form-dialog.ts, transfer-form-dialog.ts
   Example: LoanFormResult { success: boolean; count: number }

2. Manual Confirm Receipt/Return UI
   Pattern: Non-QR confirmation for loans and transfers
   Files: loans.ts, transfers.ts (row context menus)
   Example: manualConfirmReceipt(loan) → confirmReceipt API call

3. Pagination & Filtering Fix
   Pattern: Reset page index when filters change
   Problem: Filters didn't reset pagination → wrong page shown
   Solution: onFilterChange() sets pageIndex = 0
   Files: loans.ts, transfers.ts applyFilters() methods

4. finalize() Loading State Pattern
   Pattern: Track completion of batch operations
   Problem: Partial success tracking in multi-item creation
   Solution: Manual counter for success/completion tracking
   Files: loan-form-dialog.ts, transfer-form-dialog.ts createLoan()

5. takeUntilDestroyed
   Pattern: Automatic subscription cleanup
   Benefit: No manual unsubscribe() needed
   Usage: service$.pipe(takeUntilDestroyed(this.destroyRef)).subscribe()
   Files: All components with service subscriptions

6. effect() with allowSignalWrites
   Pattern: Reactive filtering with signals
   Usage: Constructor effect re-applies filters when service data changes
   Option: allowSignalWrites: true allows signal updates in effect
   Files: loans.ts, transfers.ts constructor

7. QR Code Pattern
   Pattern: Print-scan workflows for fast operations
   Components: Display QR (print/download), Scan QR (paste data)
   APIs: GET /loans/{id}/qr, POST /loans/scan-qr, POST /loans/{id}/confirm-receipt
   Files: loan-qr-dialog.ts, transfer-qr-dialog.ts

KEY INSIGHTS
================================================================================

Recent Architectural Shift:
- From promise-based to signal-based state management
- From manual subscription cleanup to takeUntilDestroyed()
- From QR-only to hybrid QR + manual confirmation
- From filter-unaware pagination to reset-on-filter pagination
- From RxJS operators (finalize) to explicit completion tracking

Best Practices Established:
- Always use OnPush change detection
- Inject services with inject() function
- Use signals for component state
- Use computed for derived state
- Use effect() only for side effects with allowSignalWrites
- Always clean up subscriptions (takeUntilDestroyed)
- Reset pagination on filter changes
- Emit typed result objects from dialogs
- Support both automated and manual confirmation flows

Canonical Examples to Copy:
- loans.ts — Filtering, pagination, list architecture
- transfers.ts — Status-based workflows, filtering
- loan-form-dialog.ts — Multi-item creation, batch tracking
- loan-qr-dialog.ts — QR display and scanning
- loan.service.ts — Service pattern with signals

Design System Highlights:
- Dark/light theme via CSS variables
- Tailwind utility classes for layout
- Angular Material for components
- Theme-aware color system (no hardcoded colors)
- WCAG AA compliance by default
